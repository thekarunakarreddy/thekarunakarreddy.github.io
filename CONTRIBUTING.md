# Portfolio Website - Contributing Guidelines

## Thank you for your interest in contributing!

This is a personal portfolio website project. While this is primarily a personal project, suggestions and feedback are welcome.

## How to Contribute

### Reporting Issues
If you find any bugs or have suggestions for improvements:

1. Check if the issue already exists in the GitHub Issues
2. If not, create a new issue with:
   - Clear description of the problem or suggestion
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Enhancements
For feature suggestions:

1. Create an issue with the "enhancement" label
2. Describe the feature and its benefits
3. Explain the use case
4. Consider implementation complexity

### Code Contributions
While this is a personal portfolio, if you have improvements to suggest:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improvement-name`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Create a pull request

## Development Guidelines

### Code Style
- Use consistent indentation (2 spaces)
- Follow semantic HTML practices
- Use modern CSS (Grid, Flexbox)
- Write clean, readable JavaScript
- Comment complex logic

### Testing
- Test on multiple browsers
- Verify responsive design
- Check accessibility compliance
- Validate performance metrics

### Documentation
- Update README if necessary
- Document any new features
- Maintain code comments

## Questions?

Feel free to open an issue for any questions or clarifications.
